# 10x 双向协作义务 — 各岗位族支撑与采纳

> **原则**：其他岗位族 **必须支撑 10x**，且 **必须利用 10x 产出优化本季业务产出**。  
> **关联**：[10x-team-assessment-framework.md](./10x-team-assessment-framework.md)

---

## 1. 义务类型

| 类型 | 定义 | 验收 |
|------|------|------|
| **支撑（Support）** | 评审、联调、场景、数据、排期按承诺交付 | 完成率记入 **支撑达标率** |
| **采纳（Adopt）** | 将 10x 产出写入 **本岗位当季产出字典** 并登记 handoff_id | 条数 ≥ **N** 记入 **采纳达标率** |

---

## 2. 各岗位族义务表

| 岗位族 | 每季最低采纳 N | 必须支撑（示例） | 必须采纳（示例） |
|--------|----------------|------------------|------------------|
| **RF01 算法** | 1 | 领域微调排期、评测基建 | release 引用 10x benchmark |
| **RF02 工程** | 2 | API/Agent 框架、联调 | 合并工作流组件、changelog |

| **PL 负责人** | 1（本线） | 联合评审、资源承诺 | 本线 KPI 含 10x 衍生项 |

## 3. 产品线绑定（支撑优先级）

| PL | 对 10x 的支撑优先级 | 说明 |
|----|---------------------|------|
| P5 | 高 | 金融/ToB 领域主战场 |
| P2 | 高 | 工作流与多模态 |
| P6 | 高 | 模型/评测 |
| P1 | 中 | 内容/垂直场景 |
| P4 | 中 | 算力与视频 |
| P3 | 低（探索降敏） | 科研向，采纳 N 可为 0 但须纪要说明 |

---

## 4. 业务门禁（叙事，非 HRIS 自动执行）

- **CPO 叙事**：某 PL **当季 10x 采纳 = 0** → 不宜单独批准该线 **算法 HC 增加**（需联合复盘）。  
- **10x + CPO 共担**：D3b 低且某 RF 采纳 = 0 → 强制双人复盘会。

---

## 5. 红线

1. 禁止 10x 独立发布与客户产品线冲突的 **影子 App**。  
2. 禁止「贴标签不交付」：采纳条目须可链接至功能/API/文档。  
3. 探索期 PL（P3）可无采纳，但须 **书面说明** 并降敏 W8/W11。
