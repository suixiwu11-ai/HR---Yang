> **文档性质**：基于 STRIDE 方法论（战略能力 × 产品线 × 岗位族 × 双通道指标）的**咨询产品化建议**，供德勤中国人力资本、战略与数字化相关团队参考。

## 摘要

德勤中国若将「战略人效核算」做成**可复用咨询产品与轻量 SaaS/交付工作台**，不宜复制单一 AI 公司的 Demo，而应把 STRIDE 已验证的方法论升级为 **「战略对齐的人力投资决策引擎」**：服务 **AI 原生企业、转型中的科技与先进制造、跨国企业中国区** 三类客户，与德勤 **人力资本趋势、技术趋势、数智化白皮书** 叙事同频，并嵌入现有 **HC / S&O / Technology / Risk** 项目售卖的 **诊断—设计—实施—运营** 链条。

建议产品代号（示例）：**ALIGN** — *Aligned Labor & Investment in Growth*（战略人力投入与增长对齐引擎），或纳入德勤既有 IP 命名体系。

## 1. 为何德勤中国适合做这件事

### 1.1 与德勤公开战略叙事的契合

| 德勤对外主题（2025–2026） | 对本产品的含义 |
|---------------------------|----------------|
| 《全球人力资本趋势》：人类×机器、价值创造、动态协同 | 人效不应是「砍人」，而是 **变革节奏下的人力投资组合** |
| 《技术趋势》：AI 融入核心系统、人才与架构协同 | 需要 **C1 模型能力 / C4 平台商业化 / C3 产品化** 三维核算，而非传统 HR 仪表盘 |
| 跨国企业在华数智化白皮书：架构/合规/AI/上云焕新 | 客户同时面临 **组织、合规、数字化** 三条线的人力重配 |
| 德勤自身：AI 赋能人才、GenAI 学习计划 | **对内先行试点** 可增强对客户的说服力（「我们自己用」） |

### 1.2 与德勤中国业务结构的契合

德勤中国的收入与交付通常横跨：

- **人力资本咨询**（组织设计、薪酬、HR 转型、劳动力分析）  
- **战略与运营**（战略规划、运营卓越、并购整合）  
- **技术咨询**（ERP/HCM、数据平台、AI 工程化）  
- **风险与合规**（内控、数据安全、上市合规叙事）

「战略人效」天然是 **跨 Practice 的楔子产品**：  
- HC 提供组织与指标语言；  
- S&O 提供战略能力与投资组合框架；  
- Technology 提供数据管道与工具；  
- Risk 提供披露口径、假设分级与免责治理。

### 1.3 市场缺口（德勤可占据的位置）

| 现有供给 | 缺口 | 德勤机会 |
|----------|------|----------|
| 传统 HRIS / People Analytics | 偏运营与合规，弱战略对齐 | 做 **「战略季」人效」** 而非日报表 |
| 纯 AI 效能工具（Copilot 用量） | 偏工具层，弱业务线/能力 ROI | 做 **能力×业务线×岗位** 三维 |
| 一次性组织诊断 PPT | 不可持续、难审计 | 做 **季度快照 + 证据层** 的可复用工作台 |
| 客户自建 Excel | 政治化、口径不一 | 做 **中立第三方方法论 + 工作坊** |

## 2. 从 STRIDE 到德勤产品：什么该保留、什么该升级

### 2.1 应保留的方法论内核（德勤 IP 化）

以下来自 STRIDE 项目已沉淀、可写入德勤 **方法论手册 + 交付模板**：

1. **战略能力层（可配置 C1–Cn）**  
   - 不固定为 AI 公司的 6 能力；按行业/客户战略地图定制（如：「全球交付」「合规」「核心产品」「中台」「生态」）。  
2. **业务线 / 价值流层（P1–Pn）**  
   - 对应 BU、产品、区域、解决方案线；支持 **explore / grow / harvest** 生命周期，避免探索期被误杀。  
3. **岗位族层（8±2 类）**  
   - 与客户职级体系映射；**成本系数为相对权重**，不冒充薪酬数据。  
4. **双通道指标**  
   - **披露通道**（财报、招股书、监管）与 **代理通道**（发布节奏、项目里程碑、调研）硬隔离。  
5. **证据层（SourceEvidence）**  
   - 每条关键指标可挂来源、页码、可信度；适合德勤 **审计思维与客户汇报**。  
6. **分角色叙事**  
   - CHRO/HRBP、业务负责人、CFO、CEO：**同一引擎，不同 Shell**（Executive 不展示「岗位末位排名」等政治敏感项）。  
7. **预警规则引擎（W1–Wn）**  
   - 投入↑产出↓、主投能力 ROI 偏弱、收入占比与投入占比杠杆背离等。  
8. **季度配置快照（QuarterConfig）**  
   - 支持「战略季」复盘与变革项目里程碑对齐。

### 2.2 德勤必须升级的部分（相对作品集 Demo）

| STRIDE 作品集约束 | 德勤产品应具备 |
|-------------------|----------------|
| 单客户、公开数据推演 | **多客户租户**、客户授权数据接入 |
| 无 RBAC | **项目级权限**（德勤项目组 vs 客户核心组） |
| SQLite 轻后端 | 可对接 **客户 Snowflake/国内云数仓、Workday/金蝶 HCM** |
| 无鉴权 | **SSO、操作日志、数据驻留策略** |
| 固定 MiniMax 能力字典 | **行业加速器**（AI 原生 / 制造 / 金融 / 跨国中国区） |
| Markdown/PDF 报告 | + **董事会一页纸、工作坊 Facilitation Kit、SAP/Excel 导出** |

## 3. 产品定位与命名建议

### 3.1 定位一句话

> **帮助企业在「战略变革季」把人力投入解释清楚、投对位置、并向董事会交差——用可审计的「战略能力 ROI」取代口水战。**

### 3.2 与德勤服务线的包装方式

建议作为 **「人力资本 + 战略 + 数字化」联合 offering**，而非单一 HC 小工具：

| 包装层级 | 内容 |
|----------|------|
| **诊断（2–4 周）** | 战略能力地图工作坊 + 指标字典 + 基线季度快照 |
| **设计（4–8 周）** | 分摊规则、预警阈值、角色视图、与 OKR/战略地图对齐 |
| **实施（8–16 周）** | 数据管道、工作台部署、培训 HRBP/FP&A |
| **运营（可选）** | 每季复盘 Facilitation、指标维护、变革跟踪 |

### 3.3 目标客户细分（德勤中国优先序）

| 优先级 | 客户类型 | 典型痛点 | 德勤切入点 |
|--------|----------|----------|------------|
| **P0** | 上市/拟上市 **AI 与硬科技企业** | 研发密集、业务线多、投资人问「人效」 | 能力 ROI + 披露/代理双通道 |
| **P1** | **跨国企业中国区** | 全球战略与本土编制脱节 | 与「数智化在华」白皮书联动 |
| **P2** | **传统行业 AI 转型**（金融、汽车、消费） | AI 专班与业务条线抢人 | 中台分摊与变革季门禁 |
| **P3** | 德勤 **内部** | GenAI 学习与组织变革 | 对内试点案例 |

## 4. 德勤版产品架构建议

### 4.1 逻辑架构（在 STRIDE 四层上扩展）

```
L0 交付与商业化层
    行业加速器 | 项目模板 | 德勤方法论版权 | 报价与工时模型
L1 体验层
    Executive | CFO | CHRO/HRBP | 业务线负责人 | 德勤顾问（Project Admin）
L2 分析层
    CalcEngine | WarningEngine | ScenarioEngine（编制模拟）| ReportBuilder
L3 领域层
    StrategicCapability | ValueStream | JobFamily | MetricDefinition
    QuarterConfig(snapshot) | AllocationMatrix | Engagement（项目）
L4 数据与证据层
    SourceEvidence | DisclosedFeed | ProxyFeed | ClientHRISFeed（授权）
L5 集成层
    Workday / 金蝶 / 用友 / 数据中台 | 德勤 Omnia/知识库（可选）
```

### 4.2 与德勤现有资产的衔接

| 德勤资产 | 衔接方式 |
|----------|----------|
| 《全球人力资本趋势》 | 每年更新 **默认能力权重** 与高管问卷 |
| 《技术趋势》 | AI 原生客户默认 C1/C2/C4 指标包 |
| 跨国企业数智化白皮书 | 「合规焕新」→ C6 类能力；「AI 焕新」→ C1/C3 |
| HC 组织设计 / 职级体系项目 | 输出直接导入 **JobFamily 与分摊矩阵** |
| S&O 战略地图项目 | 输出导入 **StrategicCapability 字典** |
| Technology 数据项目 | 建设 **L5 集成层** |

### 4.3 技术路线（务实）

| 阶段 | 建议 |
|------|------|
| **MVP（内部+1家标杆客户）** | 基于 STRIDE 栈（Web + 关系库），多租户逻辑先以 **项目 ID** 隔离 |
| **Scale** | 国内云部署、SSO、审计日志；指标配置后台化 |
| **Enterprise** | 与客户数仓/HCM API 对接；可选与德勤分析平台打通 |

**不建议** MVP 阶段追求全功能 People Analytics（流失预测、薪酬回归）；德勤差异化在 **战略对齐与可审计叙事**，不在 ML 花活。

## 5. 德勤交付方法论：五阶段

### 阶段 A：战略对齐（1 周）

- 产出：客户 **战略能力地图 C1–Cn**、当季 **主投能力（2–3 个）**、业务线 **stage** 标注。  
- 方法：高管访谈（复用 STRIDE `interview-script` / `summarize-interview` 模板）+ 战略文档阅读。  
- 德勤角色：**Facilitator**，中立记录，避免「销售编制」嫌疑。

### 阶段 B：指标与证据字典（1–2 周）

- 产出：`MetricDefinition` + 披露/代理分类 + 采集责任矩阵（RACI）。  
- 原则：每条指标完成 **「涨了证明哪项能力变强」** 一句话测试。  
- 德勤角色：与 **CFO 办公室 / 投资者关系** 对齐披露项。

### 阶段 C：组织投入基线（1–2 周）

- 产出：岗位族 FTE、分摊矩阵（行和 100%）、中台二次分摊规则。  
- 数据来源：HRIS 导出 + 顾问抽样校准 + 管理层确认（**标注 assumption**）。  
- 德勤角色：**HC + FP&A** 联合工作坊。

### 阶段 D：季度核算与预警（每季 1 周）

- 产出：`CalcSnapshot`、W1–Wn 预警、Executive / HRBP 双报告。  
- 仪式：**季度战略人力评审会**（德勤主持议程 + 工具演示）。  

### 阶段 E：编制与变革建议（可选）

- 产出：下季编制建议、主投能力 FTE 情景模拟（+20% / -10%）。  
- 边界：明确 **「建议非指令」**，客户决策责任在管理层。

## 6. 商业化与 GTM 建议

### 6.1 产品组合

| SKU | 内容 | 计费参考 |
|-----|------|----------|
| **ALIGN 诊断** | 基线快照 + 能力地图 + 指标字典 | 固定费（人天包） |
| **ALIGN 季度运营** | 每季填数、核算、复盘会 | 年费 / 季费 |
| **ALIGN 平台许可** | 多用户工作台 + 行业模板 | 订阅（按实体/员工规模档） |
| **ALIGN + 组织变革** | 诊断 + 组织设计 + 系统实施 | 联合 S&O/HC/Tech 大单 |

### 6.2 内部激励对齐

- **HC**：牵头方法论与 HRBP 培训，避免被 Tech 完全商品化。  
- **S&O**：拥有「战略能力字典」与董事会叙事。  
- **Technology**：拥有集成与平台交付毛利。  
- **Risk**：签署披露口径与假设分级治理。  

建议设立 **跨 Practice 的 ALIGN 产品负责人（Product Owner）**，汇报线可采用 **矩阵式**（虚线人力资本，实线数字化平台）。

### 6.3 营销叙事（中文，面向客户 C-level）

> 「在变革疲劳时代，董事会不再只问『有多少人』，而问『人是否投在战略能力上』。德勤 ALIGN 用可审计的季度闭环，把人力从成本争论升级为投资组合管理。」

可与德勤已发布的 **2026 全球人力资本趋势** 联合举办 Roundtable，首发 **AI 原生企业** 标杆案例。

## 7. 治理、合规与边界（德勤必做）

| 风险 | 德勤应对 |
|------|----------|
| 被理解为「裁员工具」 | 对外固定话术：**投资组合与战略对齐**；Executive 视图隐藏岗位末位 |
| 使用未授权薪酬数据 | 仅用 **相对系数 + 披露员工成本**；薪酬分析单独立项 |
| 中国数据出境 | 明确 **数据驻留**；敏感字段本地化 |
| 与审计业务利益冲突 | 咨询交付与审计客户 **防火墙**；工具由咨询 Technology 托管 |
| 指标造假 / 过度假设 | **假设占比水印**（>30% 报告封面提示）；证据层必填 |
| 建议被当作审计意见 | 报告页脚：**管理咨询建议，非审计/鉴证结论** |

## 11. 结论与下一步

德勤中国若做此产品，**核心价值不是再做一个 HR 仪表盘**，而是把德勤擅长的 **战略对话、证据思维、跨职能 facilitation** 产品化，抓住 **AI 变革季** 客户对「人力投资组合」的刚需。

**建议立即行动的三件事：**

1. **指定 ALIGN 产品负责人**（HC + Tech 双背景优先），立项 90 天 PoC。  
2. **选德勤内部或一家 AI/科技客户** 跑通一季「战略能力 ROI」闭环（可复制 STRIDE 五阶段）。  
3. **法务与品牌** 确认对外命名、免责声明与审计防火墙话术。

## 附录 C：垂直研究 /「10× 团队」类机制（2026 行业趋势）

上市 AI 公司公开推出的 **产业研究合伙人 / 10× 效率** 计划，本质是 **横切战略项目**，不是第七条产品线。德勤 ALIGN 类交付应增加：

| 模块 | 德勤交付要点 |
|------|----------------|
| **C7 能力层** | 可选主投「垂直产业 10×」，与 C1–C6 并列配置权重 |
| **双边闭环** | 研究方考核 **交接 + 被采纳**；业务方考核 **支撑 + 消费** 研究产出 |
| **分摊透明** | 与中台 70% 规则分列（如 50% 中台 + 50% 绑定业务线），避免「中台黑洞」争议 |
| **预警扩展** | W6–W11：投入增、采纳不增、只评审不交接等 |
| **Executive 叙事** | **10x 杠杆系数**（采纳/投入），避免仅展示研究人头 |
