## 0. 写给 HRBP 的一段话

你每个季度大概都会遇到这类场面：业务负责人说「人不够、要加 HC」，财务说「人力成本率又上去了」，CPO 问「算法到底投在海螺还是星野」——而你手里的表，往往是 **各产品线各做各的 Excel**，口径对不上，也不敢直接拿去高管会。

**STRIDE 要解决的，首先是 HR 专业问题**：本季 **人力总成本（TCOW）** 怎么走、**Rev/FTE** 和 **Labor Cost%** 哪条线异常、**编制达成** 和 **组织信号**（空缺、离职、Span）亮不亮红灯；其次才是把这些数字 **挂到战略能力 × 产品线 × 岗位族** 上，回答「加的人是不是加对了地方」。

产品默认界面是 **HR 工作台**（总览 / 成本 / 组织 / 薪酬×绩效），不是董事会营销页。给 CEO/CFO 的 **经营摘要** 与 HR 看板 **同源数据**，避免「两套故事」。

## 1. 公司环境：HRBP 在什么业务里干活

### 1.1 业务特征（与 HR 相关的部分）

该公司是典型的 **AI 原生、多产品线、研发极度密集** 组织：

- 多条 **C 端产品**（内容/社交、创作工具等）与 **开放平台 / API** 并行；  
- **基础模型与中台** 承接大量工程与算法工时，再 **二次分摊** 到各产品线；  
- **已上市**，每季有披露口径（收入、员工成本、海外占比等），财务对 **Labor Cost%** 敏感；  
- 2026 年前后公开 **10x Team** 等产业研究机制，带来 **跨线协作与「只取不交」** 的新型扯皮。

对 HRBP 而言，这意味着：**编制不是「总人数」一个数**，而是「谁、在哪条线、承担多少中台成本、对应哪类战略能力」——传统 HR 报表很难直接回答。

### 1.2 HRBP 常见的四类季度压力

| 压力 | 业务表现 | HRBP 典型困境 |
|------|----------|----------------|
| **成本可见性** | 营收波动大、研发占比高 | 说不清 TCOW 结构（固定/浮动/法定/福利）哪块在涨 |
| **编制博弈** | 多线抢 HC、中台分摊争议 | 只有「要多少人」，没有「投到哪个能力/产品线」的证据 |
| **组织健康** | 关键岗空、离职、加班文化 | 信号散落在招聘/ER 表，进不了季度复盘主屏 |
| **战略对齐** | 年报叙事 vs JD vs 业务诉求不一致 | 被质疑「HR 不懂战略」，又不敢用不可审计的 proxy |

**结论**：HRBP 需要的不是「再一张人效 PPT」，而是 **可重复打开的工作台**——先稳住 **People Analytics 口径**，再叠加 **战略编制叙事**。

## 2. HRBP 的六大痛点（为何要做 STRIDE）

### 2.1 成本：TCOW 与 Labor Cost% 说不清

- 财务问「人力成本率为什么升」，业务说「我们人多在做大事」——中间缺少 **同屏的 TCOW 走势 + 分产品线 Labor Cost% + 结构拆解**（固定/浮动/法定/福利）。  
- 披露通道（年报员工成本）与业务 proxy（条线营收、HC）若 **混在一张表**，会议里会被 CFO 直接否掉。

**STRIDE 回应**：Tab「成本」专做 TCOW 结构；Tab「总览」用 **双通道硬隔离** 展示 KPI，并标注 `disclosed` / `proxy` / `assumption`。

### 2.2 效率：Rev/FTE 与「人效」各说各话

- 产品线用 DAU、发版数；HR 用 HC；财务用人均收入——**没有统一的 Rev/FTE 或条线人效 proxy**。  
- 「人效低」常被简化成「人太多」，引发 **裁编恐慌** 或 **无效加人**。

**STRIDE 回应**：总览与对标图展示 **产品线 Rev/FTE**；战略层再算 **能力 ROI / 岗位人效指数**（在「战略核算」路由，不抢 HR 首屏）。

### 2.3 编制：计划、达成与缺口不同步

- 编制计划按线拆，实际入职/转岗滞后；**编制达成率**  rarely 与 **战略主投** 同表出现。  
- 关键岗空缺、替补周期缺位，直到业务投诉才暴露。

**STRIDE 回应**：Tab「组织」集中 **编制达成、Span、关键岗空缺、离职率**；与向导 2「组织投入 / 分摊矩阵」联动。

### 2.4 薪酬×绩效：只能做聚合，也要能讲清策略

- 高绩效人群占比 vs 高绩效薪酬占比 **长期偏离**，是 ER/薪酬团队的经典问题，但很少进入 **经营季度叙事**。  
- 必须坚守 **聚合视角**（n<5 不展示个案），避免被当成「裁员/砍绩效」工具。

**STRIDE 回应**：Tab「薪酬×绩效」仅部门/产品线聚合 + 策略提示 + 固定免责文案。

### 2.5 战略与分摊：Excel 政治

- 工程/算法 **70% 经 P6 再分摊**，各 PL 对负担体感差异大；HRBP 若不能展示 **矩阵 + audit 日志**，编制会議易失控。  
- 招聘 JD 写「算法 60%」，年报强调 C 端全球——**HR 缺少中立坐标系**。

**STRIDE 回应**：向导 2 + **能力×岗位×产品线** 三维矩阵（R-HRBP-01）；CPO 模板 + HRBP 调整留痕。

### 2.6 工具债：问数慢、预测靠拍脑袋

- 临时问「上季度哪条线 Labor Cost% 最高」要翻多张表；  
- 「若多招 10 个工程到 P2，下季成本率多少」没有 **透明情景**，只能会后补模型。

**STRIDE 回应（数字化）**：  
- **Insight Copilot（Chat-to-BI）**：口径与图表同源，回答带 **字段引用与来源类型**；  
- **ForecastEngine / 调薪沙盘**：编制增量、主投权重调整、调薪比例 → **区间 + 假设清单**（可解释，非黑箱 ML）。

## 3. 「战略人效」在 HR 工作中的位置

### 3.1 不是替代 TCOW，而是叠加上去

| 层级 | HRBP 关心的问题 | 典型指标 |
|------|-----------------|----------|
| **People Analytics（首屏）** | 钱与人是否健康 | TCOW、Labor Cost%、Rev/FTE、编制达成、组织信号 |
| **战略编制（第二层）** | 人与主投方向是否一致 | 能力 ROI、分摊矩阵、岗位人效指数、stage 预警 |
| **横切项目（第三层）** | 10x 等是否被产线用起来 | 支撑/采纳、协作分（见 [10x-team-program.md](./10x-team-program.md)） |
| **经营摘要（对外）** | CEO/CFO 一页结论 | 披露杠杆、主投能力摘要（与上同源） |

传统「人均收入」或单一 DAU **无法** 解释中台、探索期产品线（如 Audio）或 ToB 线——但 HRBP **仍应先站稳第一层**，再向 CPO/CEO 解释第二层。

### 3.2 本文对「战略人效」的定义（HRBP 可用版）

> 在某一战略季，在 **可审计的 TCOW 与编制事实** 之上，把人力投入映射到 **战略能力（C1–C7）× 产品线（P1–P6）× 岗位族（RF01–RF09）**；用披露与代理产出检验 **主投是否带来可观测结果**，并支撑 **下一季编制、协作与情景预测**。

关键词是 **「可审计」** 与 **「可重复」**——不是给个人排名，而是给 **组织投资决策** 提供中立底稿。

### 3.3 与 STRIDE 产品的关系

**STRIDE** = HRBP 的 **季度工作台** + 战略核算引擎 + **问数 / 预测** 能力。  
业务价值体现在：**编制会、季度复盘、与业务/财务对齐** 时少扯皮、少返工；Executive 报告是 **同一快照的摘要导出**，不是另一套数。

## 4. 预期效果：HRBP 做之前 vs 做之后

### 4.1 日常与季度节奏

| 场景 | 做之前 | 做之后（预期） |
|------|--------|----------------|
| **季中业务催 HC** | 口头估人数 | 打开组织 Tab + **情景预测**（+N FTE 对 TCOW / Labor Cost% 的影响） |
| **财务问成本率** | 事后拼表 | 总览 KPI + Copilot：「本季 Labor Cost% 同比？」并带 **披露/proxy 标注** |
| **编制会** | 分摊扯皮 | 出示 **分摊矩阵 + audit** + 主投能力权重 |
| **季度复盘材料** | PPT 拼凑 | 导出 **HRBP 详版报告**（含来源附录）；Executive 版一键同源生成 |
| **与 10x / 产线协作** | 微信扯皮 | 协作义务表：支撑项、采纳登记（战略 Tab / v1.2 模块） |

### 4.2 能力清单（HRBP 可感知）

1. **一眼看懂本季人力盘子**：TCOW 四 KPI + 走势 + 条线对标。  
2. **拆得清成本结构**：固定/浮动/法定/福利 + 预算偏离标签。  
3. **管得住组织信号**：编制达成、空缺、离职、Span（proxy 亦标注来源）。  
4. **讲得清薪酬策略一致性**：聚合视角下的高绩效薪酬偏差提示。  
5. **问得快**：Chat-to-BI 标准问法覆盖成本、编制、战略、预警。  
6. **算得明下季**：编制增量 / 调薪 / 主投权重调整 → 区间与假设。  
7. **上得去战略会**：能力×岗位×产品线矩阵 + 证据层，**不必另做一套数给 CEO**。

### 4.3 对其他角色的影响（简述，非主叙事）

| 角色 | 从 HRBP 工作台中获得什么 |
|------|--------------------------|
| **CPO** | 产品线 stage、W4 linkage、主投权重共定模板 |
| **CFO** | 披露通道单独；HCE、收入 vs 员工成本（**不替代** HR 的 TCOW 叙事） |
| **CEO** | `/executive` **一页摘要** + 免责；无岗位末位排名 |
| **产品线负责人** | 本线人效卡、10x 采纳登记（只读/协作） |

**原则**：HRBP 先自用工作台；高管材料是 **导出视图**，不是产品首页。

## 5. HRBP 季度工作流（建议）

```text
季初（对齐）
  → 向导 1：确认主投能力权重、各 PL stage、披露/代理字段
  → 与 CPO 锁定模板 + audit

季中（监控）
  → 工作台 Tab1–4 周更 / 月更 proxy
  → Copilot 回答业务/财务临时问题
  → 组织 Tab 跟踪编制达成与空缺

季末（复盘）
  → 向导 2–3：FTE、分摊、产出与证据
  → CalcEngine 核算 + W1–W5 预警
  → 情景预测：下季编制方案 A/B
  → 导出 HRBP 详版 → 生成 Executive 摘要（同源）

专项（按需）
  → 10x 联合评审、Handoff/采纳登记
  → 薪酬×绩效一致性回顾（聚合）
```

## 6. 数字化能力：HRBP 为什么要在意

| 能力 | 业务价值 | 信任机制 |
|------|----------|----------|
| **Insight Copilot** | 把「帮我拉一下数」从 2 天缩短到 2 分钟 | 仅答字典白名单指标；展示 citation；高 assumption 拒绝瞎答 |
| **ForecastEngine** | 编制会前预演「加人/调薪/改主投」后果 | 乐观/基准/保守三档 + 假设清单 + 水印 |
| **调薪沙盘** | 与财务对齐「增量 TCOW / Labor Cost% Δ」 | 滑块联动 CalcEngine，非独立黑箱 |

这些是 **HR Tech / People Analytics 产品能力**，与「会不会做战略映射」同等重要；作品集评审也应看到 **Chat-to-BI 与预测**，而非只有 ROI 大屏。

## 7. 10x Team：HRBP 需要知道的边界

10x 是 **横切项目（X10）**，不是第 7 条产品线。HRBP 相关点：

- **编制与分摊**：10x 人力建议 **50% P6 + 50% 绑定 PL**，与工程/算法「70% 经 P6」**分列展示**，避免中台黑洞争议。  
- **协作**：各岗位族有 **支撑 + 采纳** 义务；未采纳可触发 W8 等预警，用于 **复盘会** 而非自动裁人。  
- **在 UI 中的位置**：**不占用** HR 工作台默认四 Tab；在「战略核算 / 10x」模块中维护（见 PRD v1.2）。

详见 [10x-team-program.md](./10x-team-program.md) · [collaboration-10x-obligations.md](../02-requirements/collaboration-10x-obligations.md)。

## 8. 边界：HRBP 可以向业务承诺 / 不应承诺

**可以承诺**

- 口径一致、来源可追溯的 **季度人力洞察**；  
- 战略季 **编制与分摊讨论** 的中立底稿；  
- 情景 **试算**（带假设与水印）。

**不应承诺**

- 自动裁员、个人绩效排名、薪酬调整指令；  
- 替代 HRIS 的入转调离与薪酬发放；  
- 未经标注的「精确内部编制」或伪造披露数据；  
- Copilot **编造** 未在快照中的指标。
