# 协作约定

## 沟通工具
- 主沟通: 微信群（即时沟通）
- 文件共享: Git 仓库（本工作区）
- 规则: 群里不丢文件、不讨论长文档，所有留档内容进仓库 Markdown 文件

## 同步节奏
- 每日: 群里轻量同步
- 关键节点: 选题确认、技术方案、演示准备
- 截止前: 待确认（根据截止日期倒推）
- 飞书工作流编排: 待讨论（是否引入飞书做更正式的项目管理）

## 决策机制
- 范围分歧: 投票制，少数服从多数，无否决权
- 技术分歧: 以可验证性为准——能做出 demo 的方案优先
- 时间冲突: 按 deadline 倒推，截止时间优先

## 文档约定
- 所有交付物使用中文
- 临时草稿用 `temp_` 前缀
- 跨项目复用资料放入 `共享知识/`
- 黑客松项目专属资料放入 `项目/2026-05-黑客松项目/`
