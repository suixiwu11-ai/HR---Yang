1. Overview

我们在解决什么问题

角色核心痛点现状
HRBP 说不清 TCOW、编制、人效与主投是否一致 Excel + 口水战
HRBP / CPO 分摊与能力 ROI 被质疑 缺三维矩阵 + 来源分级
CPO 模型-产品不同频 缺 stage 预警与 linkage
CEO / CFO 要一页披露对齐结论 需与 HR 工作台同源，非另一套黑箱

方案（一句话）

STRIDE = HRBP 人力洞察与战略编制工作台（默认首页） + 战略能力×产品线×岗位族核算引擎 + Insight Copilot（Chat-to-BI） + 季度预测与情景沙盘 + Executive 同源摘要（子路由，非营销落地页）。

假设案例：以 MiniMax 公开业务结构（多产品线、中台 P6、10x Program、上市披露口径）为推演载体； 对外产品名不出现品牌，作品集与评审材料中可说明假设来源。

核心公式（保留 v1.1）

投入_i = FTE_i × 13周 × 系数_i × 分摊%
人效指数_i = Σ(产出 × 战略权重) / 投入 × 100
TCOW = Σ 投入（聚合）
Labor Cost% = TCOW / 营收（分披露/proxy 通道）
Rev/FTE = 营收 / HC

4.1

Insight Copilot — Chat-to-BI

面向 HRBP/CPO 的口径一致问答，不是通用聊天机器人。

  • MVP：规则 + 模板 → 预定义查询 → NL 总结
  • LLM（可选）：仅润色，生成前必须命中 allowed_metrics 白名单
  • 回答结构：结论 → 数字 → 图表锚点 → 字段来源 → 跳转
  • 拒绝：assumption>30% 或缺证据 → 不给确定数
「本季 TCOW 同比？」 「哪条 PL Labor Cost% 最高？」 「主投 C2 的能力 ROI？」 「W4 影响哪条线？」

POST /api/copilot/ask

4.2

ForecastEngine — 预测与情景

透明公式季度前瞻，非黑箱 ML（作品集可解释性优先）。

情景输入输出
编制增量+N FTE × RF × PL下季 TCOW Δ、Labor Cost% Δ
主投权重调整C1–C7 权重变更ROI 重算、W2/W3 预判
营收敏感度营收增速 ±X%Rev/FTE、HCE 区间
调薪沙盘调薪比例 × 人群增量 TCOW（Tab3 联动）

输出：乐观 / 基准 / 保守 三档 + 假设清单 + 水印。

POST /api/forecast/scenario

5. 范围

MVP（v1.2）与 Out of scope

✓ MVP 包含

  • HRBP 四 Tab（总览 / 成本 / 组织 / 薪酬×绩效）
  • TCOW / Rev/FTE / Labor Cost% / 编制达成
  • Copilot ≥12 标准问法 + 来源追溯

  • 向导 1–3 + CalcEngine + W1–W5

✗ 不做

  • 真实 HRIS / 薪酬系统对接
  • 个案级绩效、未经审计的 LLM 编造指标
  • 纯投资者关系式营销首页
  • 多租户鉴权（MVP 无登录）
