## 1. 项目是什么（公开信息摘要）

MiniMax 于 2026 年 5 月公开 **「10x Team」** 合作计划，定位为 **「产业研究合伙人」** 机制：

- 邀请金融、工业软件、游戏引擎、芯片设计、生命科学等领域的顶尖专家  
- 专家参与 **定义问题、共建评测与工作流**，再由模型 **反向驱动行业变革**  
- 目标：把已在 **编程领域** 出现的 **「10 倍效率跃迁」** 复制到更多产业  
- 合作形式：**全职研究员** 或 **Fellowship**（周期通常 ≥4 个月）  
- 资源：无限 Token、GPU、数据库与数据集；成果默认可发表；Fellowship 也可获期权  

**在 STRIDE 模型中的归类**：**横切项目 X10**，不是产品线 P7。

## 2. 与 C1–C7、P1–P6 的关系

| 维度 | 关系 |
|------|------|
| **战略能力** | 主绑 **C7 垂直产业 10×**；并强关联 **C1/C2/C4**（模型、多模态、商业化） |
| **产品线** | **不拥有** 独立 P 线收入；通过 **领域 Pod** 绑定 P1–P5（见下表） |
| **中台 P6** | 10x 人力默认 **50% 分摊记 P6**（模型/评测基建），**50% 记绑定 P 线** — 与工程算法「70% 经 P6」规则 **分列** |
| **岗位族** | 新增 **RF09：10x 产业研究**；其他 RF 对 X10 有 **支撑+采纳** 义务 |

## 3. 对整体战略的影响

### 3.1 正面

| 战略意图 | 业务效果 |
|----------|----------|
| **差异化垂直** | 相对纯 C 端聊天竞品，建立 **领域评测与工作流** 壁垒 |
| **ToB 叙事** | P5/API 获得 **行业方案包**，缩短销售与交付论证 |
| **品牌与人才** | 顶尖专家 + 论文，强化「研究型 AI 公司」 |
| **模型飞轮** | 领域数据与评测反哺 C1/C2，提升发布质量 |

### 3.2 风险（须在考核与预警中管理）

| 风险 | 缓解机制 |
|------|----------|
| 与 P1–P4 **抢算力与算法排期** | 主投权重共定；10x 杠杆系数进 CEO 视图 |
| **研究空心化**（有报告无产品） | 10x **D3b 被采纳度**；W8/W10 预警 |
| **只取不交**（产线要成果不支撑） | RF **协作分**；W9/W11 |
| **影子产品** | 红线：禁止未经 P 线验收的对外 App |
| Fellowship 管理成本 | 4 个月里程碑 + Handoff 强制 |

## 4. 双向协作原则（业务层）

```text
           支撑（评审/联调/场景/数据）
    各 RF / PL ──────────────────────► 10x Team
           ◄──────────────────────
           采纳（路线图/API/评测/案例进本季产出）
```

- **10x 考核** 含 **被采纳度**（见考核框架）  
- **其他岗位** 每季至少 **N 项** 10x 产出写入本岗位产出字典（见协作义务表）  
- **人效指数** 可乘 **协作修正系数**（协作差则当季指数最高打 85 折，详见考核框架）
